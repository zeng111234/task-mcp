# AGENTS.md

## 工作流（Harness）

本项目使用三阶段工作流，由 Hook 强制执行：

### 阶段 1：analyze（分析）
- 只能读文件，不能写
- 理解任务、读代码、制定方案
- 自动进入下一阶段

### 阶段 2：code（编码）
- 可以写文件、改代码
- 写完后跑测试验证

### 阶段 3：verify（验证）
- 跑测试/构建
- 展示结果给用户
- **必须等用户确认后才能继续**
- 用户说"合格" → 回到 code 阶段
- 用户说"不合格" → 回到 code 阶段修复

### 规则
1. 每次完成一个功能后，跑测试进入 verify 阶段
2. verify 阶段必须调用 `checkpoint` 工具展示结果
3. 等用户确认后再继续
4. 不要跳过验证阶段

### 模式
- **smart**（默认）：验证通过自动继续，失败才弹窗
- **manual**：每次都弹窗确认
- **auto**：全自动，不弹窗

用 `set_mode` 工具切换模式。
